On a Linux workstation, the route command takes a long time before printing out the routing table. Which of the following errors does that indicate?

B. 라우팅 테이블의 라우터 중 하나를 사용할 수 없으므로 ARF-D(자동 라우터 오류 감지 메커니즘)가 시간 초과를 기다립니다. 
C. 실수로 기본 라우터가 두 개 이상 있을 수 있습니다. 이 경우, 네트워크에서 기본 라우터를 선택하여 하나의 라우터를 기본값으로 선택해야 합니다. 
D. LKRD(Linux Kernel Routing Daemon)가 실행되고 있지 않으므로 init 스크립트 또는 systemd 장치를 사용하여 시작해야 합니다. 
E. DNS 확인은 기본적으로 라우터와 대상 이름을 확인하려고 할 때 라우트로 작동하지 않을 수 있으며 시간 초과가 발생할 수 있습니다.
Explanation/Reference: Linux 워크스테이션에서 route 명령은 라우팅 테이블을 인쇄하는 데 오랜 시간이 걸립니다. 다음 중 어떤 오류를 의미합니까?

QUESTION 9(Q 9)
What is true about the Hop Limit field in the IPv6 header? 
A. The field is not changed during the transport of a package.

D. Each router forwarding the packet decreases the field’s value. 
E. For multicast packages, the field’s value is always 1. 

Correct Answer: D
Section: (none)
Explanation: 
A. 패키지로 운송하는 동안 필드가 변경되지 않습니다.

D. 패킷을 앞으로 이동시키는 각 라우터는 필드 값을 줄입니다. 
E. 멀티캐스트 패키지, 필드 값은 항상 1입니다.

IPv6의 홉 제한 필든 IPv4의 라이브 시간 필드를 대체하며, 이 값은 각 전달 노드에서 1씩 감소하고 0이 되면 패킷이 삭제되어 짐. 이는 패킷이 네트워크를 무한정 순환하지 않도록 할 때 유용함. 왜냐하면 데이터 로드를 지원할 수 없는 경우 정체, 속도 및 충돌이 발생할 수 있기 때문.
